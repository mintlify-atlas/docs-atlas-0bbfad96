## Creating Events

Tap the "+" button or long-press a date to create an event.

### Event Creation Flow

**1. Choose Poster**
Select who's posting the event:
- Your personal account
- Any group where you're an admin
- Society accounts (if committee member)

Tap your profile picture in the header to change.

**2. Add Cover Image (Optional)**
Make your event stand out:
- **Choose from Templates**: Pre-designed covers for different event types
- **Upload Photo**: Use your camera roll
- **Edit Image**: Crop and adjust

<Info>
Events with cover images get 4x more RSVPs than those without.
</Info>

**3. Event Details**

**Name** (Required)
Clear, descriptive title:
- "Computer Science Society Pub Quiz"
- "Football Training Session"
- "Year 2 Study Group - Library"

**Date & Time** (Required)
- Start date and time
- End date and time (defaults to 1 hour after start)
- All-day toggle for deadlines/multi-day events

**Location** (Optional)
Choose from:
- **University buildings**: Auto-complete with room numbers
- **Popular venues**: Local bars, clubs, restaurants
- **Custom location**: Any address
- **Online**: Virtual event with meeting link

**Further Location Instructions**:
- "Meet outside main entrance"
- "Room 3.14 in Portland Building"
- "Ask for the Chapter group at the bar"

**Description** (Optional)
Provide event details:
- What to expect
- What to bring
- Dress code
- Parking information
- Contact details

**4. Recurring Events**

Create events that repeat:

**Frequencies:**
- Daily
- Weekly (same day each week)
- Custom Weekly (select multiple days)
- Bi-weekly
- Monthly

**End Options:**
- Never (continues indefinitely)
- After X occurrences
- On specific date

**Custom Weekly Example:**
Football training every Tuesday and Thursday:
- Select Tuesday and Thursday
- Set different times for each day
- Set end date (e.g., end of term)

<Warning>
Recurring events create separate calendar entries for each occurrence. Editing one doesn't affect others.
</Warning>

**5. Visibility & Capacity**

**Visibility:**
- **Public**: Anyone at your university can see and RSVP
- **Group Only**: Only visible to members of the posting group
- **Invite Only**: Only invited users can see the event

**Capacity** (Optional):
Set maximum attendees:
- Shows "X/Y attending"
- RSVP closes when full
- Waitlist option

**6. Review & Create**

Review all details and tap "Create Event".

The event is:
- Added to your calendar
- Posted to the group (if group event)
- Shared with invited users
- Discoverable in event search (if public)

## Technical Details

### Event Data Model

```swift
struct CalendarEvent {
    let id: UUID
    let title: String
    let type: CalendarActivityType
    let startTime: Date
    let endTime: Date
    let location: EventLocation?
    let description: String?
    let imageURL: String?
    let createdBy: UUID
    let maxAttendees: Int?
    let confirmedAttendees: [UUID]
    let isPublic: Bool
    let tickets: [EventTicket]
}
```

### Event Storage

Event cover images are stored:
```
https://images.chapteruni.com/event_cover_images/{EVENT_ID}/{IMAGE_ID}.jpg
```

Template images:
```
https://images.chapteruni.com/event_cover_images/templates/{category}/{id}.avif
```
